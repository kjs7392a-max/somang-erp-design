## DB 구조 (Supabase: bnlybtvdqyfxmbrcvcnv)

### ENUM 타입
```sql
draft_status:    pending | in_progress | approved | held | rejected
approval_action: pending | approved | held | rejected
```

### 테이블 구조

| 컬럼 | 타입 | 제약 |

| name | TEXT | NOT NULL UNIQUE |

| created_at | TIMESTAMPTZ | NOT NULL DEFAULT now() |

#### draft_approval_steps
| 컬럼 | 타입 | 제약 |
|---|---|---|
| id | UUID | PK |
| draft_id | UUID | NOT NULL, FK → drafts(id) |
| approver_id | UUID | NOT NULL, FK → profiles(id) |
| order_index | INTEGER | NOT NULL CHECK >= 1 |
| action | approval_action | NOT NULL DEFAULT 'pending' |
| comment | TEXT | - |
| acted_at | TIMESTAMPTZ | - |
| created_at | TIMESTAMPTZ | NOT NULL DEFAULT now() |

### RLS 정책

| 테이블 | 정책명 | 명령 | 조건 |
|---|---|---|---|
| draft_approval_steps | steps_insert | INSERT | 기안자 본인 + draft status=pending |
| draft_approval_steps | steps_select | SELECT | super_admin OR global_viewer OR 결재자 OR 기안자 |
| draft_approval_steps | steps_update | UPDATE | approver_id=me AND action=pending AND is_current_approver(draft_id) / WITH CHECK: approver_id=me |

### 트리거 & 함수

| 이름 | 대상 | 동작 |
|---|---|---|
| `trg_sync_draft_status` | draft_approval_steps (AFTER UPDATE) | 결재 action 변경 시 drafts.status 자동 동기화 |
| `trg_guard_super_admin` | profiles (BEFORE UPDATE) | is_super_admin 승격 방지 |
| `is_current_approver(uuid)` | RLS 헬퍼 | 현재 사용자가 결재 차례인지 확인 |
| `can_access_draft(uuid)` | RLS 헬퍼 | 기안 접근 권한 확인 |
