## 이번 세션 복기 — 무엇을 잘못했나

### 사건 요약
생체인식 로그인이 "은행앱처럼 빠르게" 작동하고 있었음.
세션에서 "첫 시도 실패", "10초 지연" 문제를 보고 원인을 잘못 진단 → 멀쩡한 구조를 연쇄적으로 파괴.

### 실제 원인 vs 잘못된 진단

| 증상 | 실제 원인 | 잘못한 행동 |
|---|---|---|
| 첫 시도 항상 실패 | `baba76f`에서 origin 로직 잘못 수정 | `prefetchAuthOptions` 제거 (무고한 코드를 범인으로 지목) |
| 10초 지연 | origin 불일치로 인한 재시도 + cold start | `router.push` 전환, ping 도입, profile 주입 등 과잉 수정 |
| 홈 로딩 느림 | 위 문제들의 연쇄 결과 | `injectProfile`, `WebAuthnWarmup` 등 불필요한 레이어 추가 |

### 진짜 문제는 이것이었다
- `37649d4`에서 origin 버그를 이미 고쳤음
- **거기서 멈췄어야 했음**
- 대신 prefetch를 없애고, ping으로 대체하고, router.push로 바꾸고, injectProfile 추가하는 등 구조를 계속 뜯어고쳐 원본보다 나빠짐

## 절대 건드리지 말아야 할 구조

### 1. `prefetchAuthOptions` 메커니즘
```
useState 초기화 → prefetchAuthOptions() → OPTIONS fetch 즉시 시작 (warm-up + 병렬)
useEffect → authenticate() → prefetch 결과 재사용 → 대기 없이 지문 다이얼로그
```
- 이것이 "은행앱 느낌"의 핵심
- 절대 제거하거나 ping으로 대체하지 말 것
- `_prefetchedOptions` 모듈 변수 구조 유지

### 2. `window.location.href = ROUTES.home` (생체인식 성공 후)
- PWA에서 service worker가 캐시에서 즉시 제공 → 전체 리로드도 빠름
- `router.push`로 바꾸면 오히려 AuthContext 상태 불일치로 스피너 발생
- **생체인식 성공 후에는 반드시 `window.location.href`**

### 3. `finally { setLoading(false) }` in useWebAuthn
- 성공/실패 모두 loading 해제해야 함
- "성공 시 loading 유지해서 배경 보여주기" 같은 트릭 쓰지 말 것

### 4. BiometricLockScreen 즉시 렌더링
- `initiallyRegistered && !showLoginForm` → 즉시 BiometricLockScreen
- 파란 로딩 배경 같은 중간 단계 추가하지 말 것

### 5. WebAuthn challenge 쿠키 (path: `/api/auth/webauthn/authenticate`)
- OPTIONS와 verify가 같은 path 쿠키를 공유
- path 변경, sameSite 변경, 쿠키 방식 변경 금지

## 현재 생체인식 아키텍처 (정상 상태)

```
[login/page.tsx]
useState(() => {
  if (registered) prefetchAuthOptions()  ← 렌더 즉시 OPTIONS fetch 시작
  return registered
})
useEffect → authenticate()  ← prefetch 결과 재사용

[webauthn-client.ts]
prefetchAuthOptions() → _prefetchedOptions = fetch(OPTIONS)
authenticateBiometric() → await _prefetchedOptions → json → startAuthentication → verify
                                                             ↓
                                              sessionStorage에 profile 캐시 저장

[useWebAuthn.ts]
authenticate() → authenticateBiometric() → setSession() → window.location.href

[AuthContext.tsx]
fetchProfile() → sessionStorage 캐시 확인 → 있으면 Supabase 쿼리 생략
               → 없으면 profiles + global_viewers 병렬 조회 (Promise.all)

[로그아웃]
signOut() → sessionStorage.setItem("logged_out","1") → router.push("/login")
login/page.tsx → "logged_out" 확인 → initiallyRegistered = false (자동 생체 건너뜀)
```

## 파일별 역할 요약

| 파일 | 역할 | 주의사항 |
|---|---|---|
| `webauthn-client.ts` | prefetch, 인증, 등록 클라이언트 | `_prefetchedOptions` 변수 구조 유지 |
| `useWebAuthn.ts` | 인증 훅, 세션 설정 | `window.location.href` 유지, `finally` 유지 |
| `login/page.tsx` | 로그인/생체 UI | useState에서 prefetch, 즉시 BiometricLockScreen |
| `authenticate/route.ts` | WebAuthn OPTIONS + verify API | challenge 쿠키 path 변경 금지 |
| `AuthContext.tsx` | 세션·프로필 상태 관리 | fetchProfile 캐시 로직 유지 |
| `Providers.tsx` | 전역 컨텍스트 | WebAuthnWarmup 추가하지 말 것 |
